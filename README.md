# lazy-moa

> Risk-aware LLM proxy: smart routing + conditional multi-model committee + context compression + experience learning.
>
> Point any AI agent's `base_url` to lazy-moa — it routes to the cheapest capable model, only opens a multi-model committee for high-risk questions, compresses long contexts, and learns from every decision.

## Quick Start

### Install & Configure

```bash
npx github:haichen1985/lazy-moa
```

Open `http://localhost:8080` → fill in your API keys → done.

Or use CLI wizard:

```bash
npx github:haichen1985/lazy-moa setup
# Auto-detects Ollama and env vars, just paste API keys
```

### Connect Your Agent

Change one line — your agent's `base_url`:

```python
# LangGraph / LangChain / CrewAI
llm = ChatOpenAI(base_url="http://127.0.0.1:8080/v1", api_key="none", model="auto")
```

```bash
# Claude Code
export ANTHROPIC_BASE_URL=http://127.0.0.1:8080
```

```bash
# Cursor: Settings → Models → OpenAI Base URL

# API Key: none
# Model: auto
```

```python

```


## What It Does

| Feature | Effect |
|---------|--------|
| **Smart routing** | 90% of requests → cheapest model, saves 80%+ cost |
| **Conditional committee** | High-risk questions → multi-model parallel + aggregate |
| **Lazy MOA** | Try one model first → Judge checks → escalate only if uncertain |
| **Context compression** | Trim tool logs + summarize old messages, save 60% tokens |
| **Experience learning** | Records decisions, reuses best strategy for same task type |
| **Semantic memory** | Embedding-based recall of past conversations |
| **Adaptive prompts** | System prompt scales with task complexity |
| **Cost control** | Daily budget + MOA limit + auto-degrade |

## How It Works

```
Agent request
  ↓
① Score input (keyword-based, zero LLM cost)
  ↓
② Recall relevant memory (embedding similarity)
  ↓
③ Compress context (trim tool logs, summarize old turns)
  ↓
④ Check experience (has this task type been solved well before?)
  ├── Yes → reuse best strategy → return
  └── No → score-based routing
       ├── High risk → committee (parallel multi-model)
       └── Normal → single model
            → Judge checks quality
            ├── Confident → return
            └── Uncertain → escalate to committee
  ↓
⑤ Record experience (SQLite)
⑥ Store memory (embeddings + SQLite)
  ↓
Return OpenAI-compatible response
```

## Configuration

Config is auto-generated by the setup wizard. For manual editing:

```yaml
# ~/.lazy-moa/config.yaml
models:
  cheap:
    base_url: https://api.deepseek.com/v1
    api_key: ${DEEPSEEK_API_KEY}
    model: deepseek-chat
  strong:
    base_url: https://api.openai.com/v1
    api_key: ${OPENAI_API_KEY}
    model: gpt-4o

tiers:
  C0: cheap    # simple chat
  C1: cheap    # routine coding
  C2: strong   # complex analysis
  C3: strong   # high-risk (architecture/legal/medical)

committee:
  members: [cheap, strong]
  aggregator: strong
  trigger_threshold: 0.7
```

Any OpenAI-compatible endpoint works: DeepSeek, OpenAI, Anthropic, Groq, SiliconFlow, DashScope, Ollama, vLLM, etc.

## API Endpoints

| Endpoint | Description |
|----------|-------------|
| `POST /v1/chat/completions` | OpenAI-compatible proxy (main entry) |
| `GET /v1/models` | List available models |
| `GET /health` | Health check |
| `GET /stats` | Experience engine stats |
| `GET /memory` | Semantic memory stats |
| `GET /` | Web setup page (if not configured) |

## Project Structure

```
lazy-moa/
├── src/
│   ├── index.ts          # Entry point
│   ├── config.ts         # Config loading + env var resolution
│   ├── scorer.ts         # 4-dimension risk scoring (pure keywords, zero LLM cost)
│   ├── llmClient.ts      # LLM API client (fetch) + embeddings + fallback
│   ├── judge.ts          # Answer quality judge (Lazy MOA)
│   ├── experience.ts     # Experience learning engine (SQLite)
│   ├── moa.ts            # Multi-model committee (parallel + aggregate)
│   ├── memory.ts         # Semantic memory (embeddings + L0-L3 layers)
│   ├── compressor.ts     # Context compression
│   ├── cacheControl.ts   # Anthropic prompt cache injection
│   ├── setup.ts          # Interactive CLI setup wizard
│   ├── server.ts         # Hono HTTP server + OpenAI proxy + web config
│   └── web/setup.html    # Web configuration page
├── package.json
├── tsconfig.json
├── Dockerfile
├── config.example.yaml
└── README.md
```

## Dependencies

- `hono` — HTTP server
- `@hono/node-server` — Node.js adapter for Hono
- `better-sqlite3` — SQLite for experience + memory
- `js-yaml` — Config file parsing
- `@inquirer/prompts` — Interactive CLI setup

Node 18+ required (uses native `fetch`).
