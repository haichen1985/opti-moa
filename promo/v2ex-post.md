# [开源] OptiMoa — 让 LLM API 省钱又聪明的智能路由代理

## 一句话

OpenAI 兼容的 LLM 代理：简单问题走便宜模型，复杂问题自动开多模型委员会，还会越用越聪明。

## 为什么做这个

用 GPT-4o / Claude / DeepSeek 的时候，发现两个问题：

1. **浪费钱** — "今天天气怎么样"和"设计一个分布式系统"走同一个模型，前者白烧钱
2. **质量不稳** — 单个模型有时候会瞎编，特别是高风险问题（法律/医疗/架构）

市面上的方案要么太重（LangChain 全家桶），要么太简单（纯转发）。我想要一个**轻量但聪明**的代理。

## 核心能力

### 1. 智能路由（省钱）
- 零 LLM 调用的风险评分引擎（纯关键词，<1ms）
- C0 闲聊 → 便宜模型（DeepSeek/本地）
- C1 简单 → 便宜模型
- C2 中等 → 中端模型
- C3 复杂/高风险 → 强模型

### 2. MOA 委员会（提质量）
- 高风险问题自动并行调 3 个模型
- 聚合器综合 3 个回答，取最优
- 不是每次都开委员会——只在风险评分超过阈值时触发

### 3. Judge 质量判定（自适应）
- 边界情况（不确定该用便宜还是贵的）先用便宜模型
- Judge 评估回答置信度，不够就自动升级到委员会
- **异步执行，不增加用户延迟**

### 4. 经验引擎（越用越聪明）
- 每次请求记录：任务类型、用了哪个模型、Judge 评了多少分
- 下次同类任务自动选历史最优模型
- SQLite 存储，零外部依赖

### 5. 上下文压缩 + 记忆召回
- 长对话自动摘要压缩，省 token
- 相关历史对话自动注入（向量召回）

## 接入方式

任何 OpenAI 兼容的 Agent 框架，改一行 base_url 就行：

```python
# 原来
client = OpenAI(base_url="https://api.openai.com/v1")

# 现在
client = OpenAI(base_url="http://127.0.0.1:8080/v1")
```

支持 Cursor、AutoGen、LangChain、CrewAI、Hermes 等。

## 技术栈

TypeScript + Hono + better-sqlite3，总共 ~1500 行代码，零框架依赖。

## 快速开始

```bash
npx github:haichen1985/opti-moa
# 交互式配置向导，填入你的 API Key 就行
```

## 诚实说明

- 目前评分引擎是关键词匹配，不是 ML（攒够 500 条数据后可以训练 LightGBM 路由器）
- 委员会模式会增加延迟（3 个模型并行 + 1 个聚合），适合非实时场景
- 经验引擎需要积累一定数据才能体现效果
